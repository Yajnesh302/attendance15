How a new Contract Period starts — data flow
Admin enters bid date + picks Vendor per Category
→
Old ContractPeriod → Status=Closed, EndDate set
→
New ContractPeriod row inserted → Status=Active
All active Employees with that Category
→
Old EmployeeEngagement → EndDate set, EndReason=ContractEnd
→
New EmployeeEngagement row — IsCarriedOver=TRUE, PrevEngagementId=old Id
From next working day → Attendance.ContractPeriodId = new period
→
Old period's attendance is sealed — never touched again

1Employees.ID → ∞EmployeeEngagements.EmpID Full career history — every chapter

1ContractPeriods.Id → ∞Attendance.ContractPeriodId Which period this attendance day belongs to
